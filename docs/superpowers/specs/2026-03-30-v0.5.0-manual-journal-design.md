# v0.5.0 Manual Journal (分開帳) — Design Spec

## Overview

Phase 2 첫 번째 서브그룹: `freee manual-journal` CRUD 5개 커맨드 구현.
기존 deal 커맨드 패턴을 따르되, 차변/대변 이중 입력 모드를 추가한다.

## Commands

### 1. list

```bash
freee manual-journal list --from 2026-03-01 --to 2026-03-31
freee manual-journal list --account-item-id 123 --entry-side debit --all
freee manual-journal list --adjustment only
```

**Flags:**

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--from` | string | | 시작일 (YYYY-MM-DD) |
| `--to` | string | | 종료일 (YYYY-MM-DD) |
| `--entry-side` | string | | `debit` or `credit` |
| `--account-item-id` | int64 | | 勘定科目 ID 필터 |
| `--partner-id` | int64 | | 거래처 ID 필터 |
| `--adjustment` | string | | `only` (결산만) or `without` (결산 제외) |
| `--min-amount` | int64 | | 최소 금액 |
| `--max-amount` | int64 | | 최대 금액 |
| `--limit` | int | 50 | 페이지당 건수 |
| `--offset` | int | 0 | 오프셋 |
| `--all` | bool | false | 전체 자동 페이지네이션 |

**Table Output (ManualJournalRow):**

| Column | Source | Description |
|--------|--------|-------------|
| ID | `id` | 분개 ID |
| Date | `issue_date` | 발행일 |
| Adjustment | `adjustment` | Yes/No |
| Entries | computed | "2 debit / 1 credit" |
| Amount | computed | 차변 합계 |
| Description | `details[0].description` | 첫 행 적요 (truncate) |

### 2. show

```bash
freee manual-journal show 12345
```

- 위치 인자: manual journal ID (필수)
- key-value 출력: ID, Date, Adjustment, TxnNumber
- Details 섹션: 각 행의 entry_side, account_item_id, amount, vat, tax_code, description 표시

### 3. create — 이중 입력 모드

**단순 모드 (1:1 분개):**

```bash
freee manual-journal create --date 2026-03-15 \
  --debit-account-id 100 --credit-account-id 200 --amount 50000 \
  --tax-code 1 --description "월말 정산"
```

**JSON 모드 (다행 분개):**

```bash
freee manual-journal create --date 2026-03-15 --details-json '[
  {"entry_side":"debit","account_item_id":100,"amount":30000,"tax_code":1},
  {"entry_side":"debit","account_item_id":101,"amount":20000,"tax_code":1},
  {"entry_side":"credit","account_item_id":200,"amount":50000,"tax_code":1}
]'

freee manual-journal create --date 2026-03-15 --details-file entries.json
```

**Create Flags:**

| Flag | Type | Required | Description |
|------|------|----------|-------------|
| `--date` | string | Yes | 발행일 (YYYY-MM-DD) |
| `--debit-account-id` | int64 | Simple mode | 차변 勘定科目 ID |
| `--debit-account-name` | string | Simple mode | 차변 勘定科目명 (name resolve) |
| `--credit-account-id` | int64 | Simple mode | 대변 勘定科目 ID |
| `--credit-account-name` | string | Simple mode | 대변 勘定科目명 (name resolve) |
| `--amount` | int64 | Simple mode | 금액 |
| `--tax-code` | int64 | No | 세구분 코드 |
| `--description` | string | No | 적요 |
| `--adjustment` | bool | No | 결산 정리 분개 여부 |
| `--details-json` | string | JSON mode | 명세 JSON 문자열 |
| `--details-file` | string | JSON mode | 명세 JSON 파일 경로 |
| `--dry-run` | bool | No | API 호출 없이 프리뷰 |

**Validation:**
- 단순 모드와 JSON 모드는 상호 배타
- 단순 모드: `--debit-account-id` (or name) + `--credit-account-id` (or name) + `--amount` 필수
- JSON 모드: `--details-json` 또는 `--details-file` 중 하나만 지정
- 차변 합계 = 대변 합계 검증 (client-side)

### 4. update

```bash
freee manual-journal update 12345 --date 2026-03-20
freee manual-journal update 12345 --details-json '[...]'
```

- 위치 인자: manual journal ID (필수)
- 변경된 필드만 전송 (`cmd.Flags().Changed()`)
- 단순 모드 + JSON 모드 모두 지원
- `--dry-run` 지원
- 주의: details를 업데이트하면 전체 교체 (freee API 동작)

### 5. delete

```bash
freee manual-journal delete 12345
```

- 위치 인자: manual journal ID (필수)
- `--dry-run` 지원

## Design Decisions

1. **이중 입력 모드**: 단순 1:1 분개와 복잡한 다행 분개를 모두 지원. 단순 모드는 agent-friendly, JSON 모드는 power-user용.
2. **deal 패턴 답습**: 파일 구조, 출력 방식, 플래그 네이밍 모두 기존 패턴 유지.
3. **client-side 차대 균형 검증**: API 호출 전에 차변 합계 = 대변 합계 확인하여 불필요한 API 에러 방지.
4. **details 전체 교체**: update 시 details를 부분 수정이 아닌 전체 교체. freee API 동작과 일치.
