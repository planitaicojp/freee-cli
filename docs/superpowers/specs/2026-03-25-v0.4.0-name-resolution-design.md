# v0.4.0 Name Resolution Design

## Overview

v0.4.0은 `--partner-name`과 `--account-name` 플래그를 추가하여 사용자가 ID를 외우지 않고 이름으로 리소스를 지정할 수 있게 한다.

**스코프**: 이름 해결(Name Resolution) 2개 항목만. 나머지 v0.4.0 항목(금액 포맷, 한국어 라벨, --no-header, schema 커맨드)은 v0.4.1로 이동.

## Design Decisions

### ADR-4 확장

SPEC.md ADR-4의 원칙을 따르되, 부분 일치 fallback을 추가한다:
- ID 플래그와 동등한 `--<resource>-name` 플래그를 제공하여 내부에서 ID로 변환
- 5년을 써도 거래처 ID는 외울 수 없음. `--partner-id 12345`보다 `--partner-name "주식회사A"`가 실용적
- ADR-4 원문은 "완전 일치 검색"만 기술하지만, 사용성을 위해 부분 일치 fallback을 추가 (SPEC.md ADR-4도 함께 업데이트)

### 매칭 전략

완전 일치 우선 + 부분 일치 fallback:

1. 완전 일치 검색 → 1건이면 사용
2. 완전 일치 0건 → 부분 일치(contains) 시도 → 1건이면 사용
3. 0건 → 에러 (hint: `freee partner list` 안내)
4. 2건+ → 에러 (후보 목록 + hint: `--partner-id` 사용 안내)

### 캐싱

캐싱 없음. 매번 API 호출. 사용 패턴이 보인 후 추가한다.

### 상호 배타

`--partner-id`와 `--partner-name` 동시 지정 시 에러. agent-friendly CLI에서 모호한 동작은 허용하지 않는다.

## Architecture

### 새 패키지: `internal/resolve/`

```
internal/resolve/
├── resolve.go      # PartnerID(), AccountItemID()
└── resolve_test.go
```

### 함수 시그니처

```go
// PartnerID는 cmd에서 --partner-id 또는 --partner-name을 읽어 ID를 반환.
// 둘 다 지정 시 에러. 둘 다 없으면 (0, nil).
func PartnerID(cmd *cobra.Command, api *api.FreeeAPI, companyID int64) (int64, error)

// AccountItemID는 cmd에서 --account-item-id 또는 --account-name을 읽어 ID를 반환.
func AccountItemID(cmd *cobra.Command, api *api.FreeeAPI, companyID int64) (int64, error)
```

### 매칭 로직 (공통)

```
1. cmd.Flags().Changed(idFlag) && cmd.Flags().Changed(nameFlag)
   → error: "mutually exclusive"

2. cmd.Flags().Changed(idFlag)
   → return id, nil

3. cmd.Flags().Changed(nameFlag)
   → List API 호출 (페이지네이션으로 전체 취득, 기존 --all 패턴 활용)
   → 완전 일치 필터
     → 1건: return id, nil
     → 0건: 부분 일치(contains) 필터
       → 1건: return id, nil
       → 0건: error "no X found matching 'name'"
       → 2건+: error "multiple X match 'name': [list]"
     → 2건+: error "multiple X match 'name': [list]" (완전 일치도 복수 가능)

4. 둘 다 없음
   → return 0, nil (호출 측에서 required 여부 판단)
```

## Command Changes

### 플래그 추가 대상

| 커맨드 | 추가 플래그 |
|--------|------------|
| `deal create` | `--partner-name`, `--account-name` |
| `deal update` | `--partner-name`, `--account-name` |
| `invoice create` | `--partner-name` |
| `invoice update` | `--partner-name` |
| `expense create` | `--account-name` |

> **Note**: `expense update`는 현재 `--account-item-id` 플래그가 없으므로 대상에서 제외. 향후 expense update에 명세 수정 기능이 추가되면 함께 추가한다.

### 대소문자 처리

이름 매칭은 **대소문자 무시(case-insensitive)**. `strings.EqualFold` (완전 일치), `strings.Contains(strings.ToLower(...), strings.ToLower(...))` (부분 일치) 사용. 일본어/한국어 거래처명에도 영문이 포함될 수 있으므로 (예: "Google LLC").

### 플래그 명명

`--account-name`은 `--account-item-id`와 "item" 유무가 다르지만, 간결성을 우선한다. `--account-item-name`을 alias로 등록하여 일관성도 확보한다.

### 커맨드 내 사용 패턴

기존:
```go
partnerID, _ := cmd.Flags().GetInt64("partner-id")
body["partner_id"] = partnerID
```

변경:
```go
partnerID, err := resolve.PartnerID(cmd, freeeAPI, client.CompanyID)
if err != nil {
    return err
}
if partnerID != 0 {
    body["partner_id"] = partnerID
}
```

### dry-run 동작

dry-run이어도 이름 해결은 실행한다 (읽기 전용 API). 해결된 ID가 dry-run 출력에 포함되어야 사용자가 올바른 대상인지 확인할 수 있다.

## Error Handling

### 에러 메시지 형식

```
# mutually exclusive
Error: --partner-id and --partner-name are mutually exclusive
hint: use one or the other

# not found
Error: no partner found matching "주식회사X"
hint: run 'freee partner list' to see available partners

# multiple exact match
Error: multiple partners exactly match "주식회사A":
  - 주식회사A (id: 1234)
  - 주식회사A (id: 5678)
hint: use --partner-id to specify

# multiple partial match
Error: multiple partners partially match "주식":
  - 주식회사A (id: 1234)
  - 주식회사B (id: 5678)
hint: use --partner-id to specify, or use the full name
```

### Exit Codes

| 에러 | Exit Code | 의미 |
|------|-----------|------|
| mutually exclusive | 4 | 검증 오류 |
| not found | 3 | Not Found |
| multiple match | 4 | 검증 오류 |
| API 호출 실패 | 5 | API 오류 |

기존 `internal/errors/` 패턴을 따른다.
