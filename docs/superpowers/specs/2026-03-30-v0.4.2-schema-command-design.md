# v0.4.2 Schema Command Design

## Overview

`freee schema <resource> <action>` 커맨드를 추가한다. cobra 트리에서 런타임으로 플래그 메타데이터를 추출하여 JSON Schema 또는 테이블로 출력한다. AI 에이전트가 커맨드 실행에 필요한 플래그를 프로그래밍적으로 파악하고, 사람 개발자가 빠르게 참조할 수 있도록 한다.

**스코프**: `cmd/schema/schema.go` 1개 파일 + 테스트. 외부 의존성 없음.

## 1. 커맨드 인터페이스

### 사용법

```bash
freee schema                       # 전체 리소스 목록
freee schema deal                  # deal의 하위 액션 목록
freee schema deal create           # deal create의 플래그 스키마
freee schema version               # leaf 커맨드 → 바로 플래그 스키마
```

### 인자 처리

- `Args: cobra.RangeArgs(0, 2)`
- 0개: rootCmd 자식 → 리소스명 + 설명 목록
- 1개: 해당 리소스의 자식 → 액션명 + 설명 목록
- 2개: 해당 커맨드의 플래그 → 플래그 스키마

### 플래그

| 플래그 | 타입 | 기본값 | 설명 |
|--------|------|--------|------|
| `--show-global` | bool | false | 글로벌 플래그(--format, --profile 등)를 포함 |

글로벌 `--format` 플래그를 그대로 사용하여 출력 형식을 제어한다.

## 2. 출력 형식

### table 모드 (기본, 사람용)

```
Command: freee deal create
Description: Create a new deal

FLAGS:
NAME              TYPE      REQUIRED  DEFAULT  DESCRIPTION
--type            string    yes       -        거래 종류: income, expense
--date            string    yes       -        발생일 (YYYY-MM-DD)
--partner-id      int       no        -        거래처 ID
--partner-name    string    no        -        거래처名 (--partner-id と排他)
--account-item-id int       no        -        勘定科目 ID
--account-name    string    no        -        勘定科目名
--amount          int       yes       -        金額
--tax-code        int       no        -        税区分コード

GLOBAL FLAGS (--format, --profile, etc.) are omitted. Use --show-global to include.
```

리소스/액션 목록 출력 시:

```
RESOURCE     DESCRIPTION
deal         Manage deals (取引)
invoice      Manage invoices (請求書)
partner      Manage partners (取引先)
...
```

```json
{
  "command": "freee deal create",
  "description": "Create a new deal",
  "flags": [
    {
      "name": "type",
      "type": "string",
      "required": true,
      "default": "",
      "description": "거래 종류: income, expense"
    }

```

```json
{
  "commands": [
    {"name": "deal", "description": "Manage deals (取引)"},

  ]
}
```

### YAML/CSV

기존 `internal/output` 패키지를 재사용한다.

## 3. 구현 구조

### 파일 구성

```
cmd/schema/schema.go       # 커맨드 정의 + 로직 (1파일 완결)
cmd/schema/schema_test.go  # 테스트
```

### 핵심 타입

```go
type FlagSchema struct {
    Name        string `json:"name"`
    Type        string `json:"type"`
    Required    bool   `json:"required"`
    Default     string `json:"default"`
    Description string `json:"description"`
}

type CommandSchema struct {
    Command     string       `json:"command"`
    Description string       `json:"description"`
    Flags       []FlagSchema `json:"flags"`
}

type CommandListItem struct {
    Name        string `json:"name"`
    Description string `json:"description"`
}

type CommandList struct {
    Commands []CommandListItem `json:"commands"`
}
```

### rootCmd 주입

schema 커맨드는 cobra 트리 탐색을 위해 rootCmd에 접근해야 한다. `cmd/root.go`에서 등록 시 주입한다:

```go
// cmd/root.go
schemaCmd := schema.NewCmd(rootCmd)
rootCmd.AddCommand(schemaCmd)
```

`schema.NewCmd(root)` 팩토리 함수가 rootCmd를 클로저로 캡처한다.

### required 판정

`required` 필드는 cobra의 `cmd.MarkFlagRequired()` 등록 여부로 판정한다. 런타임에서만 검증하는 플래그(RunE 함수 내부에서 체크)는 `required: false`로 표시된다. 이는 cobra 메타데이터의 한계이며, 향후 어노테이션으로 보완 가능.

### 핵심 함수

```go
// rootCmd를 캡처하는 팩토리 함수
func NewCmd(root *cobra.Command) *cobra.Command

// cobra 트리에서 타겟 커맨드를 찾는다
// root → args[0] → args[1] 순으로 탐색
func findCommand(root *cobra.Command, args []string) (*cobra.Command, error)

// 커맨드의 플래그를 스키마로 변환
// LocalFlags() + InheritedFlags()에서 추출
// rootCmd의 persistent flags는 --show-global 이 아니면 제외
func extractFlags(cmd *cobra.Command, showGlobal bool, root *cobra.Command) []FlagSchema

// 커맨드의 자식 목록을 반환 (hidden 커맨드 제외)
func listSubcommands(cmd *cobra.Command) []CommandListItem
```

### 글로벌 플래그 필터링

rootCmd의 PersistentFlags (--format, --profile, --company-id, --no-input, --verbose, --no-color, --dry-run, --no-header, --fiscal-year)는 기본적으로 제외한다. `--show-global` 시에만 포함.

단, 부모 커맨드의 persistent flags 중 rootCmd가 아닌 것(예: 리소스 커맨드에 정의된 persistent flag)은 항상 포함한다.

### leaf 커맨드 처리

인자 1개인데 해당 커맨드에 자식이 없는 경우(예: `freee schema version`), 액션 목록 대신 바로 플래그 스키마를 출력한다.

## 4. 에러 처리

| 상황 | exit code | 메시지 |
|------|-----------|--------|
| 존재하지 않는 리소스/액션 | 3 (Not Found) | `unknown command "xyz". Available: deal, invoice, ...` |
| 인자 3개 이상 | 4 (Validation) | cobra 기본 에러 |

## 5. 등록

`cmd/root.go`에서 `schema.NewCmd(rootCmd)` 팩토리로 생성 후 `rootCmd.AddCommand()`.

schema 커맨드 자체는 인증 불필요 (API 호출 없음). hidden 커맨드(예: `completion`)는 리소스/액션 목록에서 제외한다.

## 6. 테스트

| 테스트 | 내용 |
|--------|------|
| `TestFindCommand` | 정상 탐색 (2 depth), 존재하지 않는 커맨드 |
| `TestExtractFlags` | 플래그 추출, 글로벌 제외/포함 |
| `TestListSubcommands` | 자식 목록 반환 |
| `TestSchemaOutput_Table` | table 형식 출력 검증 |
| `TestSchemaOutput_JSON` | JSON 형식 출력 검증 |
| `TestLeafCommand` | 자식 없는 커맨드에 인자 1개 → 플래그 스키마 |
