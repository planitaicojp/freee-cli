# v0.4.1 Usability Improvements Design

## Overview

v0.4.1은 table 출력 개선(금액 포맷, 한국어 라벨, --no-header), --fiscal-year 글로벌 플래그, resolve 패키지 리팩토링을 포함한다.

**스코프**: usability 4개 항목 + resolve 리팩토링/테스트 보강. `freee schema <resource>` 커맨드는 v0.4.2로 이동.

## 1. 금액 콤마 포맷

### 동작

table 모드에서 `int64` 타입 값을 콤마 구분 포맷으로 출력한다.

```
# Before
ID     DATE        TYPE     AMOUNT   STATUS
12345  2026-03-01  income   1234567  settled

# After
ID     DATE        TYPE     AMOUNT      STATUS
12345  2026-03-01  income   1,234,567   settled
```

### 구현

`internal/output/table.go`에 `formatValue` 함수 추가. 값의 타입이 `int64`/`int`이면 콤마 포맷 적용.

**ID/코드 필드 제외**: json 태그 이름이 `"id"`이거나 `"_id"` / `"_code"` 로 끝나는 필드는 콤마 포맷 제외. ID에 콤마가 붙으면 복사해서 사용할 수 없고, 코드 값(tax_code 등)도 콤마를 붙이면 의미가 달라진다.

```go
func formatValue(v reflect.Value, jsonTag string) string {
    tagName := strings.Split(jsonTag, ",")[0]
    isIDOrCode := tagName == "id" || strings.HasSuffix(tagName, "_id") || strings.HasSuffix(tagName, "_code")
    if (v.Kind() == reflect.Int || v.Kind() == reflect.Int64) && !isIDOrCode {
        return formatAmount(v.Int())
    }
    return fmt.Sprintf("%v", v.Interface())
}

func formatAmount(n int64) string {
    // 음수 처리 포함
    // 1234567 → "1,234,567"
    // -1234567 → "-1,234,567"
    // 0 → "0"
}
```

### JSON/YAML/CSV 출력

콤마 포맷은 **table 모드에서만** 적용. JSON/YAML/CSV는 숫자 그대로 출력 (기존 동작 유지).

## 2. 한국어 상태 라벨

### 동작

table 모드에서 status 필드의 영문 값을 한국어로 번역한다.

```
# Before
STATUS: settled

# After
STATUS: 결제완료
```

### 구현

`internal/output/labels.go` (신규 파일)에 번역 맵:

```go
var statusLabels = map[string]string{
    // deal
    "settled":   "결제완료",
    "unsettled": "미결제",
    // invoice
    "draft":     "임시저장",
    "applying":  "신청중",
    "approved":  "승인",
    "rejected":  "반려",
    "cancelled": "취소",
    "sent":      "발송완료",
    "overdue":   "연체",
    // expense
    "remanded":  "반송",
    "sending":   "발송중",
}

// StatusLabel returns the Korean label for a status value.
// Returns the original value if no translation exists.
func StatusLabel(status string) string {
    if label, ok := statusLabels[status]; ok {
        return label
    }
    return status
}
```

### 적용 조건

`formatValue`에서 `json` 태그가 `"status"`인 필드의 `string` 값에 `StatusLabel()` 적용.

### JSON/YAML/CSV 출력

번역은 **table 모드에서만** 적용. JSON/YAML/CSV는 원문 그대로 (agent/스크립트 호환성 유지).

## 3. --no-header 플래그

### 동작

`--no-header`를 지정하면 table 출력에서 헤더 행을 생략한다.

```bash
# 기본
freee deal list
# ID     DATE        TYPE     AMOUNT      STATUS
# 12345  2026-03-01  income   1,234,567   결제완료

# --no-header
freee deal list --no-header
# 12345  2026-03-01  income   1,234,567   결제완료
```

### 구현

`cmd/root.go`에 글로벌 플래그 추가:
```go
rootCmd.PersistentFlags().BoolVar(&flagNoHeader, "no-header", false, "suppress table headers")
```

`New()` 팩토리에 `Options` 구조체를 추가하여 옵션 전달:

```go
type Options struct {
    NoHeader bool
}

func New(format string, opts Options) Formatter
```

`TableFormatter`와 `CSVFormatter`는 `Options.NoHeader`를 참조하여 헤더 행 스킵. `cmdutil.IsNoHeader(cmd)` 헬퍼를 추가하여 각 커맨드에서 `output.New(format, output.Options{NoHeader: cmdutil.IsNoHeader(cmd)})` 형태로 호출.

### JSON/YAML/CSV

`--no-header`는 table과 CSV 모드에서 적용. JSON/YAML에서는 무시.

## 4. --fiscal-year 글로벌 플래그

### 동작

`--fiscal-year 2025`를 지정하면 list 커맨드에서 from/to 날짜를 회계 기간에 맞게 자동 설정한다.

```bash
# 결산월 3월인 회사
freee deal list --fiscal-year 2025
# → --from 2024-04-01 --to 2025-03-31 로 자동 변환

# 결산월 12월인 회사
freee deal list --fiscal-year 2025
# → --from 2025-01-01 --to 2025-12-31 로 자동 변환
```

### 구현

#### 글로벌 플래그

`cmd/root.go`:
```go
rootCmd.PersistentFlags().IntVar(&flagFiscalYear, "fiscal-year", 0, "fiscal year (auto-sets --from/--to based on company closing month)")
```

#### 결산월 조회

`cmd/cmdutil/fiscal.go` (신규):

```go

// Calls GetCompany API to get the company's closing month.

    // closingMonth=3: 2024-04-01 ~ 2025-03-31
    // closingMonth=12: 2025-01-01 ~ 2025-12-31
    // closingMonth=9: 2024-10-01 ~ 2025-09-30
}
```

#### Company 모델

기존 `internal/model/company.go`에 이미 `FiscalYear` 구조체가 있음 (`ID`, `StartDate`, `EndDate` 필드). 추가 모델 변경 불필요.

결산월은 `FiscalYear.EndDate`의 월을 파싱하여 추출한다.

#### 회계 기간 선택 로직

`Company.FiscalYears` 배열에 복수의 회계 기간이 존재할 수 있다 (과거 기간 포함). 요청된 `--fiscal-year` 값에 해당하는 기간을 찾는 방법:

1. `FiscalYears`를 순회하여 `EndDate`의 연도가 요청 연도와 일치하는 항목 사용
2. 일치하는 항목이 없으면 가장 최근 항목의 결산월을 기준으로 계산
3. `FiscalYears`가 비어있으면 에러 (hint: `freee company show`로 확인)

#### 인증 실패 시

`GetCompany` API 호출 시 인증 에러가 발생하면 기존 `internal/api/client.go`의 에러 핸들링이 `AuthError` (exit 2, hint: `freee auth login`)를 반환한다. 추가 처리 불필요.

#### 적용 커맨드

`--from`/`--to` 파라미터를 지원하는 list 커맨드:
- `deal list`
- `invoice list`
- `expense list`

#### 상호 배타

`--fiscal-year`와 `--from`/`--to` 동시 지정 시 에러:
```
Error: --fiscal-year and --from/--to are mutually exclusive
hint: use --fiscal-year for full year, or --from/--to for custom range
```

Exit code: 4 (검증 오류)

## 5. Resolve 패키지 리팩토링

### Named 인터페이스

```go
type Named interface {
    GetID() int64
    GetName() string
}
```

`model.Partner`와 `model.AccountItem`에 구현:
```go
func (p Partner) GetID() int64       { return p.ID }
func (p Partner) GetName() string    { return p.Name }
func (a AccountItem) GetID() int64   { return a.ID }
func (a AccountItem) GetName() string { return a.Name }
```

### 제네릭 matchByName

```go
func matchByName[T Named](name string, items []T, resource, idFlag string) (int64, error)
```

`multipleMatchError`도 제네릭화:
```go
func multipleMatchError[T Named](resource, name string, matches []T, exact bool, idFlag string) error
```

`idFlag` 파라미터로 hint 메시지의 플래그 이름을 구분 (`--partner-id` vs `--account-item-id`). `idFlag`는 `multipleMatchError`의 hint에서만 사용되며, not-found 에러에서는 `resource` 이름으로 `'freee %s list'` hint를 생성한다.

### 삭제 대상

- `matchAccountItemByName` (matchByName으로 대체)
- `multipleAccountMatchError` (multipleMatchError로 대체)

약 80~100줄 삭감.
